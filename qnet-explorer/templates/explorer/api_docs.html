QNet API Documentation

Introduction

The QNet API provides programmatic access to the QNet blockchain data and functionality. This RESTful API allows developers to interact with the blockchain, query data, and submit transactions.

Base URL

https://api.qnet.network/v1

Response Format

All API responses are returned in JSON format with the following structure:

{
  "success": true,
  "data": {
    // Response data here
  },
  "error": null
}

In case of an error:

{
  "success": false,
  "data": null,
  "error": {
    "code": "ERROR_CODE",
    "message": "Error description"
  }
}

Authentication

The QNet API offers both public endpoints (no authentication required) and private endpoints (API key required).

API Keys

To use authenticated endpoints, you'll need to:

  1. Register for an API key from the developer portal
  2. Include your API key in the request header:
X-API-Key: your_api_key_here
Note: Keep your API key secure. Do not share it publicly or include it in client-side code.

Rate Limits

Public endpoints have stricter rate limits compared to authenticated endpoints. See the Rate Limits section for details.

Request SDK

Example Usage (JavaScript)

const QNet = require('qnet-js');

// Initialize client
const client = new QNet.Client({
  apiUrl: 'https://api.qnet.network/v1',
  apiKey: 'your_api_key' // Optional
});

// Get blockchain info
client.getChainInfo()
  .then(info => console.log(info))
  .catch(err => console.error(err));

// Create and send a transaction
const wallet = new QNet.Wallet('your_private_key');
wallet.sendTransaction({
  recipient: 'recipient_address',
  amount: 10.5
})
  .then(txHash => console.log('Transaction sent:', txHash))
  .catch(err => console.error(err));

Rate Limits

To ensure fair access to the API and maintain system performance, rate limits are applied to API requests.

Endpoint Public (Unauthenticated) Authenticated
/chain 10 requests per minute 120 requests per minute
/new_transaction 20 requests per minute 300 requests per minute
/sync_chain 5 requests per minute 60 requests per minute
/state 15 requests per minute 180 requests per minute
/get_peers 15 requests per minute 300 requests per minute
All other endpoints 15 requests per minute 180 requests per minute
Note: Exceeding rate limits will result in HTTP 429 (Too Many Requests) responses. Repeated violations may lead to temporary or permanent blocking of your IP address or API key.

Error Codes

The API uses standard HTTP status codes and returns detailed error information in the response body.

HTTP Status Codes

Status Code Description
200 OK Request successful
201 Created Resource created successfully
400 Bad Request Invalid parameters or request
401 Unauthorized Missing or invalid API key
403 Forbidden API key doesn't have permission
404 Not Found Resource not found
429 Too Many Requests Rate limit exceeded
500 Internal Server Error Server error

Transaction Endpoints

POST /new_transaction

Submit a new transaction to the network

Request Body
Name Type Required Description
sender string Yes Sender's address
recipient string Yes Recipient's address
amount number Yes Transaction amount
signature string Yes Cryptographic signature of the transaction
pub_key string Yes Sender's public key
timestamp number No Transaction timestamp (default: current time)
Response
{
  "success": true,
  "data": {
    "message": "Transaction added successfully"
  },
  "error": null
}
Example
curl -X POST "https://api.qnet.network/v1/new_transaction" \
-H "Content-Type: application/json" \
-d '{
  "sender": "your_address",
  "recipient": "recipient_address",
  "amount": 10.5,
  "signature": "transaction_signature",
  "pub_key": "your_public_key"
}'
